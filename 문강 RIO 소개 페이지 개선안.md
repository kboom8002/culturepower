현재 **문강 RIO 소개 페이지**는 “소개 페이지”라기보다 **짧은 안내문 + 발간 목적 카드 + 창간사 카드** 수준에 머물러 있습니다. 하지만 SSoT 기준으로 문강 RIO 소개 페이지는 단순 브랜드 소개가 아니라, **문강이 무엇인지, 왜 필요한지, 무엇이 다른지, 어떤 구조로 읽어야 하는지, 어떤 섹션과 도구로 운영되는지, 누구를 위한 웹진인지**를 한 번에 설명하는 **정본 소개 페이지**여야 합니다. 또한 이 페이지는 사용자가 **30초 안에 문강의 정체성, RIO 프레임워크, 3 Pillars, 6개 섹션, 실용 도구 구조**를 이해하도록 설계되어야 합니다. 

문강 RIO의 핵심 정의도 분명합니다. 문강은 일반 기사형 웹진이 아니라, **문화강국 관련 질문을 구조화하고, Reform / Implementation / Outcomes 프레임으로 정리하며, 기사·리포트·원페이저·캘린더·데이터·뉴스레터로 배포하는 정책형 Media SSoT 서브 허브**입니다. 또 문강의 차별점은 **정책 설계 + 현장 증명 + 실무 도구 제공이 동시에 작동한다는 점**, 그리고 **읽는 매체가 아니라 입법·행정 지원 도구 성격을 가진다는 점**입니다.

따라서 이 페이지는 아래처럼 전면 재구성하는 것이 맞습니다.

# 1. 현재 페이지의 핵심 문제 진단

## 1) 정보량이 너무 적고, 문강의 정체성이 약합니다

현재 화면에는

* 제목
* 짧은 소개
* 문강 RIO란?
* 발간 목적
* 창간사/편집장 글
  정도만 있어, 문강의 가장 중요한 구조인
  **RIO Framework / 3 Pillars / 6개 섹션 / Data Lab / Delivery Strategy / Newsletter / 활용 대상**이 빠져 있습니다. SSoT는 이 요소들을 소개 페이지의 필수 섹션으로 명시합니다. 

## 2) 문강이 “왜 필요한가”가 약합니다

현재의 “발간 목적” 3카드는 방향은 맞지만, SSoT가 요구하는 핵심 메시지인

* 정책 설명에서 끝나지 않고 설계안을 제시
* 현장의 사례와 사람의 목소리를 정책 승격 가능성으로 연결
* One-Pager / Data Lab / Newsletter를 묶어 실무 도구화
  가 충분히 살아 있지 않습니다. 

## 3) 문강의 구조가 보이지 않습니다

소개 페이지가 사용자를 문강의 전체 구조로 안내해야 하는데, 현재는

* 6개 섹션이 무엇인지
* 각 섹션이 어떤 역할을 하는지
* Data Lab이 왜 있는지
* 뉴스레터와 PDF/브리핑 유통 구조가 무엇인지
  가 전혀 보이지 않습니다. 

## 4) 색상과 카드 구조가 너무 옅어서 존재감이 약합니다

현재 페이지는 거의 전 구간이 **연회색 배경 + 흰 카드 + 연한 회색 텍스트**로 구성되어 있어, 정보 위계가 약하고 섹션 간 경계가 흐립니다. 법인 소개 페이지와 마찬가지로 **Hero만 강한 시각 중심**, 나머지는 **밝은 배경 + 명확한 타이포 대비 + 제한된 포인트 컬러**로 정리하는 편이 낫습니다.

## 5) 하단 Footer가 과밀합니다

현재 footer 영역은 정보량이 너무 많고, 소개 페이지 본문보다 더 길게 보입니다. 소개 페이지는 **문강 소개 본문 → CTA → 정돈된 footer 1회**로 끝나는 편이 좋습니다.

### 본문

문화정책과 문화산업, 지역문화 현장은 늘 많은 정보와 이슈를 생산하지만, 실제 정책과 제도, 행정과 현장에 바로 인용 가능한 형태로 정리되는 경우는 드뭅니다. 문강 RIO는 이 간극을 줄이기 위해 질문을 구조화하고, 정책 대안을 설계하며, 현장과 데이터를 연결하고, 실무자가 바로 사용할 수 있는 형식으로 제공하는 것을 목표로 합니다.

## Section C. 문강의 편집 프레임: RIO

### 제목

**문강의 편집 프레임: RIO**

### 설명

문강 RIO의 모든 주요 콘텐츠는
**무엇을 바꿀 것인가(Reform) → 어떻게 실행할 것인가(Implementation) → 무엇이 달라질 것인가(Outcomes)**
의 흐름을 기준으로 정리합니다. 

### 카드 3개

#### Reform

* 구조개혁 의제를 설정
* 제도·재정·거버넌스 차원의 전환 제안
* 예: 지역이 설계하는 문화정책 구조 / 문화재정의 분권과 자율성 / 예술가 생태계 지속가능성 

#### Implementation

* 대안을 실행 가능한 형태로 설계
* 최소 / 중간 / 공세 3안
* 실행 로드맵
* 예산 추계 

#### Outcomes

* 정책효과와 성과 논리를 정리
* 감사·보고·브리핑에 인용 가능한 근거 제공
* 기대효과 분석, 리스크 평가, KPI 설정 

### 디자인 메모

이 섹션은 현재 페이지에 전혀 없는데, 사실상 문강 소개 페이지의 핵심입니다.
따라서 3카드는 **가장 강한 시각 블록**으로 처리하는 것이 맞습니다.

## Section D. 문강이 지키는 3대 편집 원칙

### 제목

**문강이 지키는 3대 편집 원칙**

### Pillar 1. 국가 전략 설계 (Policy Design)

* 문화를 국가 생존 전략으로 정의
* 수동적 해설이 아닌 능동적 아젠다 제시
* 대안 3안 + 실행 로드맵 + 예산 추계 제시
* “Why this matters”를 강제 

### Pillar 2. 현장 증명 및 연결 (Field Proof)

* 중앙 하달식 정책의 한계를 넘어 현장 증명 강화
* 문화마을·생활문화 사례의 정책 승격 가능성 분석
* 기획자·예술가 인터뷰를 통한 현장 증언 확보
* 핵심 메시지: 정책은 현장에서 완성된다 

### Pillar 3. 읽히는 실용성 (Readable Utility)

* 학술적 분석보다 실무 활용성 우선
* 원페이지 요약 제공
* 지원사업 캘린더 제공
* 국회/지자체/현장 실무자의 즉시 활용 지향 

### 레이아웃

* 3열 카드
