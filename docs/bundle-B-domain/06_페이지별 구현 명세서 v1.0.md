**06\_페이지별 구현 명세서 v1.0**

**1\. 문서 목적**

이 문서는 문화강국네트워크 AI홈페이지의 각 페이지를 **무슨 역할로 설계할지, 어떤 블록 순서로 구현할지, 어떤 CMS 필드를 어디에 연결할지, 무엇을 노출하고 무엇을 숨길지**를 페이지 단위로 규정하기 위한 기준 문서다.  
즉, IA·GNB·사이트맵 명세서와 CMS 컬렉션·필드 사전을 전제로 하여, 실제 화면 구현과 운영을 연결하는 문서다. 이 정의는 hanjunho.ai 페이지별 구현 명세서의 목적 정의와 동일하다.

이 문서에서 가장 중요한 원칙은 세 가지다.

*   각 페이지는 **하나의 핵심 목적만** 가진다.
*   허브 페이지와 정본 상세 페이지를 분리한다.
*   첫 화면 안에서 이 페이지가 무엇이고, 누가 말하며, 다음 행동이 무엇인지 보여야 한다.

**2\. 공통 구현 원칙**

**2-1. 모든 페이지는 역할이 분명해야 한다**

소개, 탐색, 정답, 기사 해설, 기록, 신뢰, 전환이 한 페이지 안에서 뒤섞이면 안 된다.  
예를 들어 Home은 허브, AnswerCard 상세는 정본, Story 상세는 기사형 해설, Event 상세는 기록 정본, Join은 행동 전환 허브다. 이 원칙은 hanjunho.ai 구현 명세서의 “각 페이지는 하나의 핵심 목적만 가져야 한다”는 규칙을 그대로 적용한 것이다.

**2-2. 첫 화면 1스크린 안에서 정체성이 드러나야 한다**

사용자는 스크롤하기 전에 다음을 이해해야 한다.

*   이 페이지가 무엇을 위한 페이지인지
*   누가 말하는지
*   무슨 행동을 할 수 있는지

hanjunho.ai도 상세 페이지의 첫 화면에 질문, 3문장 답, 검수, 업데이트일을 넣고, 브리핑 상세에는 기사 출처, 요약, 핵심 쟁점을 상단에 두라고 합니다. 문화강국네트워크도 같은 수준으로 상단 구조를 강제해야 합니다.

**2-3. 허브와 정본 상세를 분리한다**

허브는 요약과 진입을 담당하고, 정본은 설명과 신뢰를 담당한다.  
이는 hanjunho.ai가 홈/클리어맵/이슈브리핑 인덱스를 허브, Answer Card 상세와 이슈브리핑 상세를 정본으로 나눈 구조와 같다.

**2-4. 상단 본문 구조가 메타보다 중요하다**

hanjunho.ai 문서 읽기 가이드는 “메타만 잘 넣으면 검색이 된다”는 생각이 틀렸고, **질문/답/요약/쟁점/역할 정의 같은 본문 상단 구조**가 더 중요하다고 설명합니다. 문화강국네트워크도 모든 핵심 페이지에서 이 원칙을 따른다.

**2-5. 삭제 작업도 구현 항목이다**

현재적 관점에서 불필요한 템플릿성 문구, 의미 없는 더미 블록, 역할이 불분명한 범용 블로그형 섹션은 구현 대상이 아니라 제거 대상이다. hanjunho.ai 구현 명세서도 “무엇을 넣을지 못지않게 무엇을 빼야 하는지가 중요하다”고 명시한다.

**4\. Home 페이지 구현 명세**

**4-1. 페이지 역할**

**질문 진입 허브 + 편집 큐레이션 허브 + 행동 전환 허브**

Home은 단체 소개 페이지가 아니다.  
사용자가 “이 사이트가 무엇을 답하는지”를 빠르게 이해하고, 문강 RIO·정답카드·행사·참여로 진입하게 만드는 허브다. 이는 hanjunho.ai 홈을 질문 진입 허브로 본 방식과 같다.

**4-2. 핵심 사용자 행동**

*   대표 질문을 통해 정답카드 또는 TopicSummary로 진입
*   편집 큐레이션을 통해 문강 RIO 대표 글로 진입
*   행사/영상/자료 아카이브로 진입
*   회원가입/뉴스레터/협력 문의로 전환

**4-3. 페이지 구조**

**1블록. Hero**

*   H1: 문화강국 담론의 공식 정답 본점
*   보조 설명: 문강 RIO, 정답카드, 행사·영상·자료·회원 참여를 한곳에서 연결하는 허브
*   대표 질문 검색창 또는 빠른 질문 4~6개

이 구조는 구성안의 홈에 웹진, 단체, 행사, 회원, 문화예술정보가 함께 있어야 한다는 요구를 “질문 진입” 중심으로 재배치한 것이다.

**18\. 공통 삭제/비공개 원칙**

*   범용 템플릿성 홍보 문구
*   질문 없이 감상문처럼만 보이는 잡문
*   검수/작성자/업데이트 정보 없는 공개 콘텐츠
*   연결 객체가 전혀 없는 고립 페이지
*   자료 없는 행사 상세
*   근거 없는 선언형 정답카드

**19\. 최종 요약**

이 문서의 결론은 다음이다.

*   **Home**은 질문 진입 + 편집 큐레이션 + 행동 전환 허브다.
*   **문강 RIO 인덱스**는 기사형 해설 허브다.
*   **Story 상세**는 질문형 기사 해설 정본 페이지다.
*   **정답카드 인덱스/상세**는 공식 질문-답 허브와 정본 페이지다.
*   **3 Pillars / TopicSummary**는 아젠다 탐색 허브와 상위 정본 허브다.
*   **행사·영상**은 기록형 아카이브 허브와 정본 페이지다.
*   **데이터·자료**는 실무 정보 허브다.
*   **네트워크**는 정체성·신뢰 허브다.
*   **참여·회원**은 행동 전환 허브다.
*   **검색**은 보조가 아니라 핵심 진입 허브다.

즉, 이 문서는 문화강국네트워크 AI홈페이지를 “단체 소개 + 웹진 + 행사 공지”의 병렬 조합이 아니라, **질문 → 정답 → 해설 → 기록 → 신뢰 → 참여** 구조를 가진 Media SSoT형 페이지 시스템으로 구현하기 위한 기준서다.
