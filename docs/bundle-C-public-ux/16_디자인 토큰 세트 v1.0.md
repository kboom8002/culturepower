**16\_디자인 토큰 세트 v1.0**

**1\. 문서 목적**

이 문서의 목적은 문화강국네트워크 AI홈페이지의 시각 시스템을 **구현 가능한 token 체계**로 고정하는 데 있다.  
즉, 디자이너가 Figma에서 쓰는 값과 프론트/Webflow 구현자가 쓰는 값, 그리고 CMS 운영자가 반복해서 마주치는 컴포넌트 규격이 모두 같은 언어를 쓰게 만드는 문서다.

이 문서는 다음을 해결해야 한다.

*   색상값을 어떤 역할 체계로 운용할 것인가
*   타이포 위계와 행간을 어떤 변수로 고정할 것인가
*   spacing, radius, shadow를 어떤 리듬으로 가져갈 것인가
*   버튼, 칩, 카드, 검색창, 요약 박스 같은 핵심 컴포넌트를 어떤 component token으로 잠글 것인가
*   문화강국네트워크만의 “문강 RIO + 정답카드 + 행사 기록 + 신뢰 + 참여” 시각 구조를 어떻게 안정적으로 재현할 것인가

이 문서는 단순한 CSS 변수 모음이 아니라, 앞선 15\_UI 컴포넌트·디자인 시스템 가이드를 실제 구현 자산으로 바꾸는 단계다. hanjunho.ai도 디자인 토큰을 UI 가이드와 별도 문서로 분리해, 시각 정책과 구현 값을 나눠 관리합니다.

**2\. 토큰 설계 원칙**

문화강국네트워크의 토큰은 아래 3층 구조로 운영한다.

**2.1 Core Token**

순수 값이다.  
예:

*   색상 hex
*   글자 크기
*   line-height
*   spacing
*   radius
*   shadow

**2.2 Semantic Token**

역할이다.  
예:

*   text-primary
*   surface-card
*   button-primary-bg
*   badge-reviewed-bg
*   surface-answer

**2.3 Component Token**

실제 UI 컴포넌트 규격이다.  
예:

*   answer-card.padding
*   story-card.title-size
*   search-bar.height
*   trust-strip.gap
*   event-card.meta-gap

이 3층 구조는 hanjunho.ai 토큰 세트의 기본 원칙을 그대로 계승한다. 실무적으로는 Core를 자주 바꾸지 않고, Semantic과 Component를 조정하는 것이 운영과 확장에 가장 쉽습니다.

**3\. 컬러 토큰**

**3.1 설계 원칙**

문화강국네트워크의 컬러는 “분위기”보다 “역할”을 먼저 표현해야 한다.  
즉,

*   Navy/Blue는 브랜드와 구조
*   Gold는 검수와 권위
*   Red는 경고와 쟁점
*   Neutral은 본문과 배경
*   Soft Blue는 질문/정답의 친절한 표면  
    에만 사용한다.

이 원칙은 hanjunho.ai의 컬러 체계와 같지만, 문화강국네트워크는 정치 플랫폼보다 조금 더 편집형·문화형 느낌이 필요하므로, Blue/Navy의 농도는 유지하되 전체 배경과 surface는 더 부드럽게 가져간다.

hanjunho.ai도 Blue는 브랜드/행동, Gold는 검수, Red는 쟁점/주의, Neutral은 본문과 배경, Light Blue는 정답과 친절함을 담당한다고 정리합니다.

**14.2 Webflow / 프론트 적용 원칙**

*   Core token은 CSS 변수로 최상위 선언
*   Semantic token은 component wrapper 또는 utility class에 매핑
*   Component token은 실제 클래스(component-card-answer, component-hero, component-chip-reviewed)에 연결
*   CMS는 텍스트/이미지/링크/상태값만 주입

이 원칙은 hanjunho.ai UI 가이드의 “컴포넌트 기준 클래스 구조”, “기본 컴포넌트 + 변형 구조”, “디자인 컴포넌트는 고정하고 CMS는 텍스트/이미지/링크만 채워 넣는 방식”과 동일하다.

**15\. 금지 토큰 / 금지 확장 규칙**

문화강국네트워크는 아래 방향으로 토큰을 늘리지 않는다.

*   neon / 형광 계열 추가
*   brand gradient 추가
*   과한 shadow variant 다량 생성
*   button style 5종 이상 추가
*   chip variant를 무제한 증가
*   page-specific color token 남발
*   hero마다 다른 radius와 다른 spacing을 쓰는 패턴

hanjunho.ai UI 가이드도 너무 진한 풀컬러 배경, 네온 계열, 과한 그림자, 슬라이더 남발, 상위 정보구조처럼 보이는 갤러리/Contact 처리를 금지합니다. 토큰 수준에서도 같은 방향을 유지해야 한다.

**16\. 최종 결론**

이 토큰 세트의 핵심은 단순합니다.  
문화강국네트워크의 디자인 토큰은 **Blue/Navy를 브랜드와 구조에, Gold를 검수와 권위에, Red를 쟁점과 정정에, Neutral을 읽기와 배경에, Soft Blue를 정답과 질문 표면에만 쓰는 역할 기반 체계**여야 합니다. 그리고 타이포는 긴 글을 읽는 플랫폼답게 가독성과 위계를 우선하고, spacing은 4px base / 8px rhythm, radius는 16~24px 중심, shadow는 매우 약하게, 버튼은 3~4종, 칩은 역할 기반, 카드와 Hero는 고정된 component token으로 운영해야 합니다. hanjunho.ai 토큰이 Core/Semantic/Component 3층 구조, color/typography/spacing/radius/shadow/button/chip/card 규격을 제공하고, UI 가이드가 역할 기반 컬러와 정보 구조 중심 시각 언어를 강조하며, 문화강국네트워크 구성안이 문강 RIO·행사·회원·자료가 함께 있는 질문형 담론 허브를 요구한다는 점을 종합하면, 이 토큰 세트는 **홍보형 웹진**이 아니라 **공식 질문형 문화 플랫폼**을 구현하는 최소 시각 인프라로 보는 것이 맞습니다.
