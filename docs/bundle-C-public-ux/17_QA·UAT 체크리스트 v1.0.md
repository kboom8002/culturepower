**17\_QA·UAT 체크리스트 v1.0**

**1\. 문서 목적**

이 문서의 목적은 문화강국네트워크 AI홈페이지 개편 작업이 완료된 뒤, 실제 공개 전에 **무엇을 어떤 기준으로 점검해야 하는지**를 최종적으로 정리하는 데 있다.  
이 문서는 세 가지 목적을 가진다.

첫째, 디자이너·개발자·편집자·운영자가 서로 다른 기준으로 “완료”를 판단하지 않도록 한다.  
둘째, 단순 오탈자 점검을 넘어서 **정보구조, UI, CMS 연결, 카피, 구조화데이터, 내부 링크, 모바일 가독성, 신뢰 구조, 전환 구조**까지 통합적으로 검수한다.  
셋째, 이번 프로젝트의 핵심 목표인 **문화강국 담론의 공식 정답 본점화, 문강 RIO의 기사형 해설 구조화, 행사/영상/자료 아카이브화, Trust Layer 가시화, AEO 정렬**이 실제 구현물에 반영되었는지 확인한다. hanjunho.ai QA/UAT 문서도 같은 3가지를 문서 목적의 핵심으로 둡니다.

이 문서는 단순 체크리스트가 아니다.  
이 문서는 **배포 승인 기준서**다. 즉, 핵심 항목이 통과되지 않으면 공개하지 않는 것을 원칙으로 한다.

**2\. QA와 UAT의 구분**

**2.1 QA의 의미**

QA는 Quality Assurance, 즉 품질 보증이다.  
디자인, 레이아웃, 링크, CMS 연결, 상태값, 구조화데이터, 반응형, 콘텐츠 일관성, 검색/필터, Feed/Sitemap 정합성 같은 **기술적·운영상 품질**을 확인한다. hanjunho.ai QA/UAT 문서도 QA를 همین 범위로 정의합니다.

**2.2 UAT의 의미**

UAT는 User Acceptance Test, 즉 사용자 수용 테스트다.  
실제 사용자가 이 사이트를 보았을 때,

*   이 사이트가 무엇을 하는 곳인지 이해되는가
*   질문을 시작할 수 있는가
*   정답이 믿을 만하게 보이는가
*   문강 RIO가 기사형 해설 허브처럼 읽히는가
*   행사와 기록이 구조적으로 참고 가능한가
*   회원가입/뉴스레터/행사 참여가 자연스럽게 이어지는가  
    를 검증한다. hanjunho.ai 문서도 UAT를 “사이트 정체성 이해, 질문 시작 가능성, 답변 신뢰감, 브리핑의 기사형 해설성” 검증으로 정의합니다.

**2.3 이번 프로젝트의 운영 원칙**

*   QA는 디자이너·개발자·CMS 운영자 중심으로 수행한다.
*   UAT는 기획자·편집자·운영자·실사용자 관점 검토자가 함께 수행한다.
*   문화강국네트워크는 Media 패밀리이므로, “기사형 해설 허브처럼 보이는가”, “편집 신뢰가 보이는가”, “행사와 자료가 기록 자산처럼 읽히는가”가 UAT에서 특히 중요하다.

**3\. 검수 범위**

이번 QA/UAT의 검수 범위는 아래와 같다.

*   Home
*   문강 RIO 인덱스
*   Story 상세 템플릿
*   정답카드 인덱스
*   AnswerCard 상세 템플릿
*   3 Pillars 허브
*   TopicSummary 상세
*   행사 인덱스 / Event 상세
*   영상 인덱스 / Video 상세
*   데이터·자료 허브
*   네트워크 허브 / 조직 소개 / Expert 상세
*   참여·회원 허브
*   검색 결과 페이지
*   전역 GNB / 푸터 / 공통 CTA / 공통 메타 / 공통 schema
*   Feed / Sitemap / indexing 운영 상태

hanjunho.ai QA/UAT 문서도 홈, 허브, Answer Card 상세, 기사형 해설 상세, 신뢰 허브, 프로필, 전역 구조, 공통 schema를 함께 검수 범위로 묶습니다. 문화강국네트워크도 같은 방식으로 범위를 잡는 것이 맞습니다.

또 문화강국네트워크 구성안상 홈, 웹진, 단체 소개, 행사/토론회/캠페인, 문화예술정보, 회원가입이 모두 필수 구조이므로, 검수 범위에서도 이 축이 빠지면 안 된다.

**4\. 검수 우선순위 체계**

체크 항목은 세 단계로 나눈다.

**P0: 공개 전 반드시 해결해야 하는 항목**

하나라도 미해결이면 공개 보류다.

예:

*   잘못된 GNB 구조
*   문강 RIO/정답카드/행사/참여 위계 붕괴
*   샘플 카드·더미 텍스트·템플릿 찌꺼기 노출
*   AnswerCard 상세가 정상 렌더링되지 않음
*   Story 상세에 Deck/Why This Matters/Reviewer 누락
*   Event 상세에 일정·요약·자료 연결 누락
*   Reviewer / UpdatedAt / Evidence 누락
*   title/meta/OG/schema 중대한 오류
*   Feed 200 실패
*   answers sitemap 부재
*   핵심 CTA 오작동

hanjunho.ai도 P0를 “잘못된 GNB, 샘플/범용 콘텐츠 공개, 상세 렌더링 실패, 구조화데이터 오류, 핵심 CTA 오작동” 같은 항목으로 정의합니다. 문화강국네트워크에 맞게 변형하면 위 항목이 핵심입니다.

**P1: 공개 가능하나 빠른 수정이 필요한 항목**

공개 후에도 가능하지만 1~3일 내 보완이 필요한 항목이다.

예:

*   일부 줄바꿈 어색함
*   카드 간 간격 불균형
*   일부 메타 디스크립션 미세 조정 필요
*   추천 카드 정확도 보완
*   모바일 간격과 칩 래핑 개선
*   일부 페이지의 OG 이미지 품질 보완

**P2: 다음 개선 배치에 반영 가능한 항목**

기능상 문제는 아니지만 완성도 향상을 위한 항목이다.

예:

*   미세 인터랙션
*   카드 hover refinement
*   이미지 크롭 통일성
*   필터 UX 고도화
*   Answer Feed 샤딩 준비

hanjunho.ai도 P0/P1/P2 삼단계 우선순위를 동일하게 운영합니다.

**5\. 검수 진행 방식**

**5.1 권장 검수 순서**

1.  전역 구조 검수
2.  홈 / 허브 페이지 검수
3.  상세 템플릿 검수
4.  CMS 데이터 검수
5.  메타·OG·schema 검수
6.  Feed / Sitemap 검수
7.  모바일·브라우저 검수
8.  UAT 시나리오 검수
9.  최종 승인

hanjunho.ai QA/UAT 문서도 거의 동일한 순서를 권장합니다. 문화강국네트워크는 Feed/Sitemap 레이어를 추가로 명시하는 것이 좋습니다.

**5.2 검수 기록 방식**

각 항목은 아래 네 가지 상태 중 하나로 표시한다.

*   \[ \] 미확인
*   \[통과\] 확인 완료
*   \[보완필요\] 수정 후 재확인 필요
*   \[보류\] 현재 공개 불가 사유 있음

이 상태 체계는 hanjunho.ai QA/UAT 문서의 형식을 그대로 따른다.

**5.3 이슈 기록 형식**

각 이슈는 아래 형식으로 기록하는 것이 좋다.

*   이슈 번호
*   페이지명
*   문제 설명
*   우선순위(P0/P1/P2)
*   수정 담당자
*   수정 완료일
*   재검수 결과

이 포맷도 hanjunho.ai QA/UAT 문서의 권장 형식을 그대로 계승한다.

**6\. 전역 구조 QA 체크리스트**

**6.1 GNB 구조**

*   상단 GNB가 다음 7개로만 구성되는가  
    문강 RIO / 정답카드 / 3 Pillars / 행사·영상 / 데이터·자료 / 네트워크 / 참여·회원
*   3 Pillars의 세부 토픽이 GNB에 직접 노출되지 않는가
*   모바일 햄버거 메뉴에서도 동일한 7개 메뉴가 유지되는가
*   현재 페이지 active 상태가 시각적으로 분명한가
*   GNB가 “협회 소개 사이트”가 아니라 “질문형 문화 플랫폼”처럼 읽히는가

hanjunho.ai QA/UAT는 GNB 항목 수 고정, 하위 pillar의 과도한 직접 노출 금지, 모바일 메뉴 일치, active 상태 명확성을 전역 구조 QA의 최우선 항목으로 둡니다. 문화강국네트워크도 동일한 원칙을 적용해야 합니다.

**6.2 푸터 구조**

*   푸터 링크가 전역 IA와 일치하는가
*   템플릿 기본 푸터(서비스/솔루션/파트너류)가 남아 있지 않은가
*   정정·문의, 개인정보처리방침, 이용정책 등이 보조 링크로 정리돼 있는가
*   푸터가 전역 구조를 다시 설명하는 수준으로 정돈되어 있는가

hanjunho.ai도 템플릿 기본 푸터 제거와 전역 정보구조 일치를 핵심 QA로 봅니다.

**6.3 전역 표기 정리**

*   “FAQ형 일반 사이트”, “협회 홍보 문구”, “서비스형 SaaS 템플릿 문구”가 남아 있지 않은가
*   문강 RIO, 정답카드, 3 Pillars, 행사·영상 등 공식 용어가 일관되는가
*   Reviewer / Evidence / UpdatedAt / Boundary / ChangeLog 표기가 일관되는가
*   날짜 표기 형식이 전역에서 일관되는가

B-SSoT 공통 원칙은 객체 중심, trust layer 가시성, 운영형 정답 인프라 사고를 요구하므로, 전역 용어와 신뢰 표기의 일관성은 필수 QA 항목이다.

**7\. Home QA 체크리스트**

문화강국네트워크 구성안은 홈에서 **웹진 중심 편집 큐레이션, 문강 RIO, 행사/토론회/캠페인 배너, 회원가입 신청 안내, 단체 소개, 문화예술정보**가 함께 보이도록 요구합니다. 따라서 홈 QA는 이 6축이 질문형 구조 안에서 잘 묶였는지를 검증해야 합니다.

**7.1 첫 화면 이해도**

*   첫 화면에서 사이트가 “문화강국 담론의 공식 정답 본점”으로 이해되는가
*   Hero H1과 서브카피가 협회 소개문이 아니라 질문형 정답 플랫폼 정체성을 설명하는가
*   검색창이 첫 화면에서 충분히 눈에 띄는가
*   검색창 플레이스홀더가 질문형 예시로 적절한가

**7.2 구조와 위계**

*   편집장/발간인 큐레이션 블록이 보이는가
*   문강 RIO 하이라이트가 대표 Story 중심으로 정리되는가
*   대표 AnswerCard가 과도하게 많지 않고 6~8개 수준으로 정리되는가
*   행사/토론회/캠페인 블록이 보이는가
*   회원가입 / 뉴스레터 / 협력 CTA가 보이는가
*   문화강국네트워크 소개와 문화예술정보 블록이 보이는가

**7.3 데이터 품질**

*   비관련 샘플 카드가 노출되지 않는가
*   임시 Story나 테스트 Event가 노출되지 않는가
*   잘못된 카테고리 카드가 섞여 있지 않은가

**7.4 기능**

*   검색창이 관련 결과 또는 허브로 정상 이동하는가
*   대표 카드 클릭 시 올바른 상세 페이지로 이동하는가
*   문강 RIO / 정답카드 / 행사·영상 / 참여 CTA가 모두 정상 작동하는가

hanjunho.ai QA/UAT도 홈 첫 화면에서 플랫폼 정체성 이해, 검색창 가시성, 대표 카드 수 조절, 신뢰 블록과 행동 CTA 노출을 핵심 항목으로 둡니다. 문화강국네트워크도 같은 수준으로 검수해야 합니다.

**8\. 문강 RIO 인덱스 QA 체크리스트**

**8.1 구조와 정체성**

*   페이지가 단순 블로그 목록이 아니라 기사형 해설 허브처럼 보이는가
*   제목과 설명문이 명확한가
*   창간사 / 발간 목적 / 스페셜 인터뷰 진입이 구조적으로 보이는가
*   정기콘텐츠 섹션이 명확히 구분되는가

**8.2 카드 품질**

*   각 Story 카드에 Section, Story Type, 제목, Deck, 작성자/업데이트 정보가 보이는가
*   카드가 “기사 링크”보다 “해설 페이지”처럼 보이는가
*   Featured Story가 잘 구분되는가

**8.3 콘텐츠 품질**

*   범용 잡문, 홍보문, 일반론이 공개되어 있지 않은가
*   K-문명, 문화정책, 지역문화, 사람과 현장 같은 핵심 질문군이 실제로 드러나는가
*   구성안이 요구한 인터뷰 / 정기콘텐츠 구조가 반영되는가

실제 문강 RIO는 질문형 장문 해설을 기본으로 삼아야 하므로, 인덱스도 최신순 블로그 목록보다 편집 허브처럼 읽혀야 합니다. 구성안의 문강 RIO 구조 요구와 실제 K-문명 글의 질문형 해설 특성을 함께 기준으로 검수해야 합니다.

**9\. Story 상세 QA 체크리스트**

**9.1 첫 화면 핵심 구조**

*   breadcrumb가 보이는가
*   Section / Story Type / Author / Reviewer / UpdatedAt이 보이는가
*   H1이 기사형 해설 제목으로 적절한가
*   Deck이 상단에서 잘 보이는가
*   Why This Matters가 상단에서 잘 보이는가
*   Based On Answers 블록이 상단 또는 상단 직후에 보이는가

**9.2 본문 구조**

*   질문 또는 문제의식이 분명한가
*   개념 정의가 있는가
*   배경/사례/정책 맥락이 구조적으로 정리되어 있는가
*   본문이 기사 복붙이 아니라 해설형 문서처럼 읽히는가
*   관련 AnswerCard가 1개 이상 연결되는가

**9.3 신뢰 구조**

*   Reviewer가 표시되는가
*   UpdatedAt이 표시되는가
*   관련 자료 / Event / Video / Archive 연결이 있는가
*   Boundary Note가 필요한 글에는 실제로 존재하는가

**9.4 데이터 정확성**

*   Story Type과 Section이 맞는가
*   Based On Answers가 실제 내용과 맞는가

hanjunho.ai QA/UAT에서 기사형 해설 상세는 breadcrumb, 라벨, 매체/작성 주체, 상단 요약, 기사 출처, 핵심 쟁점, 관련 카드 연결을 핵심으로 봅니다. 문화강국네트워크 Story도 이와 같은 수준의 구조를 가져야 합니다.

**10\. 정답카드 인덱스 QA 체크리스트**

**10.2 카테고리 품질**

*   잘못된 카테고리나 임시 태그가 공개되어 있지 않은가

**10.3 기능**

*   필터가 즉시 적용되는가
*   검색과 필터를 동시에 써도 결과가 정상인가
*   카드 클릭 시 올바른 상세 페이지로 진입하는가

hanjunho.ai QA/UAT의 카드 리스트 구조 검수 항목을 문화강국네트워크 카테고리 체계에 맞게 그대로 변환한 것이다.

*   근거 자료 섹션이 존재하는가

**12.2 TopicSummary 상세**

*   Representative AnswerCards가 보이는가

**13\. 행사·영상·자료 QA 체크리스트**

**13.3 Video 상세**

*   출연자 정보가 보이는가
*   관련 Event가 연결되는가
*   관련 Story / AnswerCard가 연결되는가

**13.4 ArchiveDocument / 자료실**

*   자료실이 단순 파일 목록이 아니라 맥락 있는 기록 허브처럼 보이는가

hanjunho.ai도 기록형 페이지는 단순 갤러리나 파일 모음이 아니라 맥락과 링크 허브처럼 보이게 해야 한다고 봅니다. 문화강국네트워크도 같은 기준이 적합합니다.

hanjunho.ai 프로필 QA도 “단순 소개 페이지처럼 보이지 않는가”, “공식 발화자 / 총괄 검수자 페이지처럼 읽히는가”를 핵심 기준으로 둡니다. 문화강국네트워크도 조직과 전문가가 신뢰 허브 역할을 해야 합니다.

**21\. UAT 시나리오 체크리스트**

hanjunho.ai QA/UAT 문서는 “일반 유권자 / 지지자 / 비지지층 / 기자·리서처” 시나리오를 사용해 사이트 정체성, 공유 가능성, 신뢰감, 구조적 참고 가능성을 검증합니다. 문화강국네트워크는 Media 패밀리에 맞게 아래 시나리오로 변환하는 것이 적절합니다.

**시나리오 1: 일반 독자**

질문: **이 사이트가 무엇을 하는 곳인지 바로 알 수 있는가?**

*   홈 첫 화면에서 문화강국 담론의 공식 플랫폼임을 이해할 수 있다
*   질문 검색 또는 대표 질문을 통해 원하는 답변으로 들어갈 수 있다
*   정답카드 상세를 읽고 “답을 들었다”는 느낌이 든다
*   문강 RIO를 읽고 “설명을 들었다”는 느낌이 든다

**시나리오 2: 문화정책·문화현장 실무자**

질문: **정책과 현장을 구조적으로 참고할 수 있는가?**

*   정답카드에서 핵심 정의와 방향을 빠르게 파악할 수 있다
*   문강 RIO에서 기사형 해설을 읽을 수 있다
*   행사와 발제문, 자료집으로 맥락을 확장할 수 있다
*   문화예술정보와 자료 허브가 실무적 도움이 된다

**시나리오 3: 신중한 외부 독자 / 회의적 독자**

질문: **신뢰할 만한 플랫폼처럼 보이는가?**

*   Reviewer와 UpdatedAt이 잘 보인다
*   근거 자료가 보인다
*   네트워크/Expert/행사 기록이 신뢰를 보강한다
*   문강 RIO가 구호나 홍보문이 아니라 설명형 문서처럼 읽힌다

**시나리오 4: 기자·리서처·협력자**

질문: **공식 참고 허브로 활용할 수 있는가?**

*   AnswerCard에서 핵심 답을 빠르게 잡을 수 있다
*   Story에서 맥락과 개념 정리를 확인할 수 있다
*   Event와 Archive를 따라가며 자료를 확인할 수 있다
*   내부 링크를 따라가며 의미 그래프를 탐색할 수 있다

**시나리오 5: 잠재 회원 / 구독자**

질문: **참여할 이유와 방법이 분명한가?**

*   회원가입의 의미가 먼저 설명된다
*   뉴스레터가 무엇을 주는지 이해된다
*   행사 참여/협력 문의 경로가 분명하다
*   참여 CTA가 콘텐츠와 자연스럽게 연결된다

**22\. 공개 승인 기준**

다음 조건이 모두 충족되어야 공개 승인 가능으로 본다.

*   P0 항목이 모두 해결되었다
*   전역 GNB 구조가 최종안과 일치한다
*   샘플 카드, 테스트 Story, 임시 Event, 더미 텍스트가 제거되었다
*   AnswerCard 상세와 Story 상세가 정본 구조로 구현되었다
*   네트워크/Expert가 신뢰 허브 역할을 수행한다
*   Event/Video/Archive가 기록 허브 역할을 수행한다
*   모바일에서 핵심 페이지의 정보 위계가 무너지지 않는다
*   title/meta/OG/schema의 중대한 오류가 없다
*   Feed / sitemap / canonical / lastmod의 중대한 오류가 없다
*   검색창, 필터, 내부 링크, CTA가 정상 동작한다

hanjunho.ai 공개 승인 기준은 “P0 모두 해결, 전역 구조 일치, 샘플 제거, Answer Card와 브리핑 상세 정본 구조 구현, 신뢰 허브 작동, 모바일 위계 유지, title/meta/OG/schema 오류 없음, 검색/필터/링크 정상”을 요구합니다. 문화강국네트워크는 여기에 Event/Archive와 Feed/Sitemap 항목을 추가하면 됩니다.

**23\. 최종 정리**

이 QA/UAT 체크리스트의 핵심은 단순합니다.  
문화강국네트워크 AI홈페이지의 공개 여부는 “페이지가 보이느냐”로 판단하면 안 됩니다. 이 사이트는 문화강국 담론의 공식 정답 본점, 문강 RIO의 기사형 해설 허브, 행사·영상·자료의 기록 아카이브, 네트워크와 참여를 연결하는 Media SSoT형 AI홈페이지로 설계되고 있습니다. 따라서 구조, 카피, CMS 데이터, 반응형, Reviewer/Evidence/UpdatedAt, Feed/Sitemap, 구조화데이터까지 함께 검증해야 합니다. hanjunho.ai QA/UAT도 같은 이유로 “예쁜지 아닌지를 보는 검수”가 아니라 “사이트가 의도한 정체성을 실제로 수행하는지 확인하는 검수”여야 한다고 정리합니다. 문화강국네트워크도 בדיוק 이 원칙으로 QA/UAT를 운영해야 합니다.
