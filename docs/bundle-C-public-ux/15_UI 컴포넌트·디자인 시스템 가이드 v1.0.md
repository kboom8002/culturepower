**15\_UI 컴포넌트·디자인 시스템 가이드 v1.0**

**1\. 문서 목적**

이 문서의 목적은 문화강국네트워크 AI홈페이지의 **시각 언어, 레이아웃 규칙, 핵심 UI 컴포넌트, 반응형 원칙, 금지 패턴**을 한 번에 고정하는 데 있다.  
이 문서는 단순 “색상표”나 “예쁜 화면 참고안”이 아니다. 이 문서는 다음을 동시에 만족시키기 위한 구현 기준서다.

*   디자이너가 페이지마다 다른 스타일을 임의로 만들지 않게 한다.
*   개발자가 Webflow/프론트에서 같은 컴포넌트를 일관된 클래스 구조로 반복 구현하게 한다.
*   문화강국네트워크 사이트를 **협회 홍보물**이 아니라 **공식 질문형 문화 플랫폼**처럼 인식하게 만든다.

이는 hanjunho.ai UI 가이드의 목적 정의를 문화강국네트워크용으로 그대로 확장한 것이다.

**2\. 디자인 시스템의 최상위 개념**

**2.1 한 줄 정의**

**문화강국네트워크의 UI는 “단체 홍보 홈페이지”보다 “공식 질문형 문화 플랫폼”처럼 보여야 한다.**

즉 사용자가 사이트에 들어왔을 때,

*   “행사 공지 게시판”을 보는 느낌보다
*   “문화강국 담론을 정리하고 설명하는 신뢰 가능한 공공 플랫폼”을 보는 느낌이 먼저 와야 한다.

이 정의는 hanjunho.ai의 “캠프 홈페이지보다 공식 질문형 정책 플랫폼처럼 보여야 한다”는 원칙을 문화강국네트워크의 Media SSoT 문맥에 맞게 옮긴 것이다.

**2.2 시각적 방향성**

문화강국네트워크의 시각 방향성은 아래 5가지를 동시에 만족해야 한다.

첫째, **공적 신뢰감**이 있어야 한다.  
가벼운 스타트업형 랜딩페이지나 과한 매거진형 연출보다, 공적 책임과 구조적 신뢰가 먼저 보여야 한다. hanjunho.ai도 “가벼운 스타트업형 UI나 광고 랜딩페이지 느낌”을 피하라고 정리합니다.

둘째, **질문형 명료성**이 있어야 한다.  
질문, 정답, 해설, 행사, 자료, 검수 정보가 한눈에 구조적으로 보여야 한다. 이는 B-SSoT의 시각 정본 객체 원칙과 연결됩니다.

셋째, **편집형 친절함**이 있어야 한다.  
관공서 문서처럼 차갑게 보이지 않으면서도, 읽기 쉽고 따라가기 쉬워야 한다. hanjunho.ai도 정책 SaaS형 명료성과 생활 밀착형 친절함의 균형을 요구합니다.

넷째, **기록 플랫폼의 차분함**이 있어야 한다.  
행사·영상·발제문·사진 기록이 쌓이는 구조이므로, 지나치게 이벤트성 컬러나 축제형 톤으로 가면 안 된다.

다섯째, **선전형 과잉 연출을 피해야 한다.**  
K-문명이나 문화강국을 상징하는 이미지를 쓴다고 해서 영웅 서사나 국가주의적 과장으로 보이면 안 된다. 실제 문강 RIO 글도 K-문명을 선언이 아니라 질문으로 다룹니다.

**3\. 시각 키워드**

문화강국네트워크 UI의 핵심 시각 키워드는 아래 7개다.

*   명료함
*   신뢰
*   정돈됨
*   설명 가능성
*   편집성
*   기록성
*   참여 가능성

이 키워드는 hanjunho.ai의 “명료함, 신뢰, 정돈됨, 설명 가능성, 생활 연결성, 검증 가능성” 키워드를 문화강국네트워크의 Media SSoT 성격에 맞게 조정한 것이다.

판단 기준은 단순하다.  
어떤 새 UI 요소가 추가될 때 이 7개 중 4개 이상과 맞지 않으면 제거 또는 수정한다.

**4\. 컬러 시스템**

**4.1 기본 원칙**

컬러는 적게 쓰고, **역할이 분명해야 한다.**  
문화강국네트워크는 색으로 시선을 끄는 사이트가 아니라, 정보 위계와 신뢰 구조를 분명히 하는 사이트다. hanjunho.ai도 “색은 장식이 아니라 의미 체계”라고 정의합니다.

**4.2 권장 컬러 구조**

**A. Primary Color — Culture Navy**

*   예시: #16324F
*   역할: 브랜드 대표색, 상단 헤더, 핵심 CTA, H1 보조 강조
*   사용 위치: 로고/브랜드명, 주요 버튼, 주요 링크 hover, 핵심 제목 보조 강조

이 색은 hanjunho.ai의 Trust Blue처럼 “브랜드와 행동 유도” 역할을 하되, 문화/기록/편집 플랫폼 특성에 맞게 조금 더 차분한 네이비 축으로 설정하는 것이 적절하다. 이것은 설계 제안이다. hanjunho.ai가 Blue를 브랜드·CTA·핵심 제목 강조에 쓰라고 한 원칙을 따르되, 문화강국네트워크의 톤에 맞춰 보정한 것이다.

**B. Core Text — Deep Ink**

*   예시: #1A2433
*   역할: 본문용 강한 텍스트, H1, 핵심 섹션 제목
*   사용 위치: 모든 주요 타이틀, 설명형 본문 상위 텍스트

**C. Editorial Gray**

*   예시: #5F6B7A
*   역할: 설명문, 메타 정보, 업데이트일, 카드 하단 메타
*   사용 위치: deck, 설명문, breadcrumb 보조 텍스트

**D. Answer Surface**

*   예시: #EEF4FB
*   역할: 정답 박스, 질문형 플랫폼의 친절함, 검색 포커스 배경
*   사용 위치: AnswerCard 요약 박스, 질문 말풍선 보조 표면, 검색창 focus 상태

이는 hanjunho.ai의 Light Blue Surface를 정답 박스와 검색창 포커스 배경에 쓰는 원칙을 그대로 응용한 것이다.

**E. Trust Gold**

*   예시: #8A6200
*   역할: 검수, 권위, 인증 신호
*   사용 위치: Reviewer badge 텍스트, 검수 칩, 검토 완료 상태
*   보조 배경: #FFF7E8

hanjunho.ai도 Gold를 “검수, 권위, 인증 신호”에만 쓰고, 메인 컬러처럼 쓰지 말라고 명확히 제한합니다. 문화강국네트워크도 동일하게 적용한다.

**F. Neutral Surfaces**

*   Background Gray: #F6F8FB
*   White: #FFFFFF
*   Border Gray: #D8E2ED
*   Soft Surface Gray: #F5F7FA

역할:

*   전체 배경
*   카드 배경
*   경계선
*   질문 말풍선 / 보조 컨테이너

**G. Semantic Colors**

*   Alert Red: #B3261E
*   Alert Surface: #FFF2F2
*   Success Green: #2E7D32
*   Success Surface: #EFFAF2

역할:

*   주의 / 오류 / 정정 / 경고
*   폼 완료 상태
*   업데이트 성공 상태

**4.3 색상 사용 요약**

*   Navy = 브랜드 / 구조 / CTA
*   Ink = 핵심 내용
*   Gray = 메타 / 설명
*   Answer Surface = 정답 / 친절한 구조화
*   Gold = 검수 / 신뢰
*   Red = 경고 / 정정 / 쟁점 주의
*   Green = 제출 완료 / 정상 상태

색상은 분위기 장식이 아니라 **정보 역할**이어야 한다. hanjunho.ai가 이 원칙을 강하게 강조합니다.

**5\. 타이포그래피 시스템**

**5.1 기본 원칙**

문화강국네트워크는 글을 많이 읽는 사이트다.  
따라서 폰트의 멋보다 **위계, 행간, 문단 리듬**이 중요하다. hanjunho.ai도 같은 이유로 폰트 선택보다 위계와 행간을 더 중요하게 둡니다.

기본 원칙:

*   제목은 강하고 짧게
*   본문은 부드럽고 길게 읽히게
*   모바일 기준 가독성을 최우선
*   문단은 길되 덩어리감 있게 분리

**5.2 권장 폰트**

*   기본 한글: Pretendard
*   보조 시스템: sans-serif

hanjunho.ai가 같은 이유로 Pretendard를 권장합니다.

**5.3 위계**

**Hero / H1**

*   44~60px
*   Bold
*   line-height 120~130%

**Section Title**

*   32~40px
*   Bold
*   line-height 125~135%

**Card Title / Subhead**

*   22~28px
*   Bold

**Body Large**

*   18~20px
*   Regular 또는 Medium
*   line-height 160%

**Body Base**

*   16px
*   Regular
*   line-height 160%

**Meta / Caption**

*   13~14px
*   Medium
*   line-height 150%

이 기준은 hanjunho.ai 위계 체계를 거의 그대로 가져오되, 문화강국네트워크의 장문 해설성을 반영해 Body Large와 Body Base 가독성을 특히 우선한다.

**5.4 강조 규칙**

*   한 문단 내 굵은 강조는 1~2개만
*   링크는 색 + hover 변화를 분명히
*   긴 문장 전체 bold 금지
*   인용문은 body보다 1단 위계로 분리

**6\. 레이아웃 시스템**

**6.1 전체 폭**

*   데스크톱 최대 콘텐츠 폭: 1200px ~ 1280px
*   긴 문서형 페이지 본문 컬럼은 더 좁게 유지

hanjunho.ai도 긴 문서형 페이지의 가독성을 위해 1200~1280px 수준을 권장합니다.

**6.2 그리드**

*   데스크톱: 12-column
*   태블릿: 8-column
*   모바일: 4-column 또는 1-column

**6.3 여백 시스템**

권장 spacing scale:

*   4
*   8
*   12
*   16
*   24
*   32
*   48
*   64
*   96

기본 규칙:

*   섹션 간격: 64px 또는 96px
*   카드 내부 패딩: 20~28px
*   작은 칩/버튼 패딩: 10~16px

이 spacing scale과 vertical rhythm 원칙은 hanjunho.ai와 동일하게 가져가는 것이 안정적이다.

**6.4 컨테이너 규칙**

*   대부분의 섹션은 하나의 큰 container 안에서 정렬
*   좌정렬 기본
*   섹션마다 정렬 기준이 바뀌지 않게 유지

hanjunho.ai도 정렬 기준이 바뀌면 사이트가 템플릿 조각처럼 보인다고 경고합니다.

**7\. 모서리, 테두리, 그림자**

**7.1 라운드**

*   칩: 14~16px
*   버튼: 14~18px
*   카드: 20~24px
*   대형 컨테이너: 24~28px

**7.2 테두리**

*   얇은 border 기본
*   Border Gray 계열 사용

**7.3 그림자**

*   매우 약하게
*   권장: 0 8px 24px rgba(16, 37, 62, 0.06)

hanjunho.ai는 정치/공공 플랫폼에서 과한 그림자가 저급해 보일 수 있다고 지적합니다. 문화강국네트워크도 같은 기준이 적절하다.

**8\. 아이콘·이미지 시스템**

**8.1 아이콘**

*   선형(line) 아이콘 위주
*   아이콘 스타일 혼합 금지
*   장식보다 의미 전달용

권장 위치:

*   검색
*   근거 자료
*   업데이트일
*   breadcrumb
*   작성/검수
*   행사 일정
*   자료 열기

hanjunho.ai도 거의 같은 위치를 권장합니다.

**8.2 이미지 원칙**

*   Hero 이미지는 감정과 인물 인지를 담당
*   카드 이미지 남용 금지
*   정답카드와 Story 카드에서는 텍스트 구조 우선
*   프로필 이미지는 실제 인물 위주
*   placeholder, 과한 스톡 이미지, 과도한 연출 컷 금지

hanjunho.ai도 정책 카드와 브리핑은 이미지보다 텍스트 구조가 중요하다고 명확히 적고 있습니다. 문화강국네트워크도 문강 RIO와 정답카드에서 같은 원칙을 적용해야 한다.

**9\. 핵심 UI 컴포넌트 가이드**

**9.1 GNB**

구성:

*   로고/브랜드명
*   전역 메뉴 7개
*   모바일 햄버거

디자인 원칙:

*   화이트 배경
*   하단 얇은 구분선
*   현재 페이지 강조는 색상 + 밑줄 또는 굵기
*   hover는 Navy 또는 Deep Ink

hanjunho.ai GNB 원칙을 그대로 계승한다.

**9.2 Hero**

역할:

*   페이지 정체성과 핵심 행동을 첫 화면에서 설명

구성:

*   소형 라벨
*   강한 H1
*   설명문
*   CTA 2~3개
*   필요 시 검색창 또는 신뢰 배지

디자인 원칙:

*   좌정렬
*   문장 길이 짧게
*   Primary CTA는 1개만
*   나머지는 Secondary 또는 Text 버튼

hanjunho.ai의 Hero 원칙과 동일하되, 문화강국네트워크는 Home Hero에 검색 진입과 문강 RIO/정답카드/행사 진입이 함께 있어야 한다.

**9.3 검색창**

역할:

*   질문 진입의 핵심 인터랙션

구성:

*   입력 필드
*   플레이스홀더
*   검색 버튼
*   보조 힌트

플레이스홀더 예시:

*   문화강국이란, K-문명이란, 지역문화 사례처럼 질문으로 찾아보세요

이는 hanjunho.ai의 자연어 예시형 플레이스홀더 원칙을 문화강국네트워크에 맞게 바꾼 것이다.

**9.4 칩(Chip)**

문화강국네트워크에서 칩은 매우 중요하다.  
문강 RIO Section, Story Type, 검수, Topic, Event Type, 자료 유형 같은 정보를 짧게 요약한다.

권장 종류:

*   Section 칩
*   Story Type 칩
*   Pillar 칩
*   Reviewer 칩
*   Event Type 칩
*   Document Type 칩

디자인 원칙:

*   pill 형태
*   색상은 역할 기반
*   한 화면에 너무 많은 칩 금지

hanjunho.ai 칩 체계를 거의 그대로 재활용하는 것이 좋다.

**9.5 버튼**

버튼은 3종으로만 운영한다.

*   Primary Button
*   Secondary Button
*   Tertiary / Text Button

버튼 문구 원칙:

*   동사 중심
*   짧고 분명하게
*   한 버튼에 한 행동만

예:

*   정답 보기
*   문강 RIO 읽기
*   관련 행사 보기
*   영상 보기
*   자료 보기
*   뉴스레터 구독
*   회원 가입하기

hanjunho.ai와 같은 원칙이다.

**9.6 Answer Card 리스트 카드**

역할:

*   질문 탐색과 빠른 스캔

권장 구조:

**11\. CMS 컴포넌트 연결 원칙**

hanjunho.ai는 “디자인 컴포넌트는 최대한 고정하고, CMS는 텍스트/이미지/링크만 채워 넣는 방식으로 가라”고 명확히 말합니다. 문화강국네트워크도 이 원칙을 그대로 적용하는 것이 가장 안전합니다.

운영 원칙:

*   Hero, 카드, 리스트, Trust Strip, Meta Strip, CTA Bar는 고정 컴포넌트로 설계
*   CMS는 텍스트/이미지/링크/상태값만 주입
*   데이터 로직과 시각 로직을 한 컴포넌트 안에 과도하게 섞지 않음
*   Featured 여부, 상태값, 칩 종류는 CMS 필드로 제어

**12\. 금지 패턴**

다음 패턴은 사용하지 않는다. hanjunho.ai의 금지 패턴을 문화강국네트워크에 맞게 거의 그대로 적용한다.

*   너무 진한 풀컬러 배경 반복
*   텍스트보다 이미지가 더 큰 카드
*   강한 네온/형광 계열 색상
*   카피보다 장식 그래픽이 더 눈에 띄는 섹션
*   슬라이더 남발
*   굵은 그림자 남용
*   하나의 섹션 안에서 정렬 기준이 2개 이상 섞이는 배치
*   행사/영상/자료를 갤러리 사이트처럼만 보이게 하는 연출
*   기업형 SaaS 랜딩페이지처럼 보이는 영문/비즈니스 카피 배치
*   국가주의적 상징을 과도하게 전면 배치하는 영웅형 Hero

마지막 금지 항목은 K-문명 글의 경고, 즉 개념을 선언이나 우월성 표현으로 오해하게 만드는 방향을 피하기 위한 추가 규칙이다.

**13\. 시각 완성도 체크 기준**

최종 산출물이 완성되었다고 판단하는 기준은 아래다.

*   홈이 질문을 시작하게 만드는가
*   문강 RIO가 블로그가 아니라 해설 허브처럼 읽히는가
*   Answer Card는 정답 페이지처럼 읽히는가
*   Story 상세는 기사형 해설 페이지처럼 읽히는가
*   Event는 기록 정본 페이지처럼 읽히는가
*   Network/Expert는 공식 발화자/검수자 페이지처럼 읽히는가
*   Trust Strip이 신뢰 허브처럼 읽히는가
*   Blue / Gold / Red의 역할이 일관되는가
*   카드와 섹션 간 여백 리듬이 일정한가
*   모바일에서도 정보 위계가 무너지지 않는가

이 기준은 hanjunho.ai의 시각적 완성도 체크 기준을 문화강국네트워크 구조에 맞게 변환한 것이다.

**14\. 최종 결론**

이 가이드의 핵심은 간단합니다.  
문화강국네트워크 UI는 **협회 홍보 페이지처럼 보여서는 안 됩니다.** 이 사이트는 질문에 답하고, 문강 RIO로 해설하고, 행사와 자료를 기록하고, 누가 쓰고 누가 검토하는지 드러내며, 참여로 이어지게 하는 **공식 질문형 문화 플랫폼**처럼 보여야 합니다. 홈페이지 구성안이 웹진 중심 큐레이션, 행사·회원·정보의 동시 운영을 요구하고, B-SSoT가 시각 객체와 Trust Layer를 핵심 구조로 보며, hanjunho.ai UI 가이드가 역할 기반 컬러, 가독성 중심 타이포, 카드형 탐색, 첫 화면 핵심 구조, 금지 패턴을 명확히 제시한다는 점을 종합하면, 문화강국네트워크 디자인 시스템도 **색은 역할 기반으로, 타이포는 읽기 기반으로, 카드는 탐색 기반으로, 상세는 정답/해설/기록이 첫 화면에 보이게, 신뢰는 골드 보조 강조로, 이미지와 장식은 절제된 방식으로** 설계하는 것이 최적입니다.
