**32\_DB·엔티티 스키마 명세서 v1.0**

**1\. 문서 목적**

이 문서는 문화강국네트워크 AI홈페이지의 **논리 DB 스키마와 엔티티 관계 모델**을 정의한다.  
목표는 퍼블릭 페이지, Admin Console, Feed/Sitemap, Observatory, Participation, Audit/Privacy를 하나의 데이터 모델로 묶는 것이다.

**2\. 설계 원칙**

*   객체 중심 설계
*   퍼블릭 객체와 운영 객체 분리
*   PII 분리 저장
*   관계형 DB + join 테이블 중심
*   모든 핵심 객체는 stable id 보유
*   audit/change/fixit은 append 중심
*   feed/sitemap은 materialized view 또는 generated table 가능

B-SSoT는 페이지가 아니라 객체/그래프 단위로 사고하라고 명시하고, AIO-PR는 Brand KG와 Answer Card, Feed, Observatory를 일관된 데이터 계약으로 묶으라고 제안한다. DB 스키마도 이 구조를 따라야 한다.

**10\. 최종 결론**

32번 문서는 문화강국네트워크 AI홈페이지를 “페이지 묶음”이 아니라 “객체·관계·운영 로그·참여 데이터”로 구현하기 위한 DB 계약이다. 핵심은 canonical content, participation, workflow, publish, graph, audit를 한 모델 안에서 분리-연결하는 것이다. 이는 B-SSoT의 객체 중심과 운영 루프, 그리고 AIO-PR의 Brand KG/Feed/Sitemap/Obs 구조를 DB 수준으로 구현하게 해준다.
