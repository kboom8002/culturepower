**34\_시드 데이터·테스트 픽스처 명세서 v1.0**

**1\. 문서 목적**

이 문서는 문화강국네트워크 프로젝트의 초기 개발, QA, UAT, 데모, Antigravity 병렬 구현을 위해 필요한 **seed data / fixture set**를 정의한다.

AIO-PR는 3개월 런칭 패키지 최소 스택으로 KG min + AC 12~18 + Feed + Map + KPI baseline 을 제시하고, 문화강국네트워크 구성안은 웹진, 행사, 회원, 자료, 문화예술정보를 함께 요구한다. 따라서 시드 데이터도 퍼블릭 콘텐츠와 운영 데이터, 참여 데이터가 함께 있어야 한다.

**2\. Seed Data 설계 원칙**

*   실제 IA를 대표해야 함
*   Topic/Answer/Story/Event/Archive/Participation이 모두 있어야 함
*   trust fields 포함
*   workflow 상태 다양성 포함
*   feed/sitemap 대상과 비대상 모두 포함
*   QA용 edge case 포함

**3\. 최소 Seed Set**

**3-1. TopicSummary**

권장 4~6개

*   문화강국
*   K-문명
*   지역문화
*   문화정책
*   행사·기록
*   참여·회원

**3-2. AnswerCard**

권장 12~18개  
대표 질문 예:

*   문화강국이란 무엇인가
*   K-문명이란 무엇인가
*   문화강국네트워크는 어떤 단체인가
*   회원이 되면 무엇을 할 수 있는가
*   발제문은 어디서 볼 수 있는가
*   지역문화는 왜 중요한가
*   문강 RIO는 어떤 웹진인가
*   행사 참여는 어떻게 하나

AIO-PR의 최소 스택이 AC 12~18개를 권장하므로, 문화강국네트워크도 최소 이 정도 세트가 있어야 Feed/KPI/UX 테스트가 가능하다.

**3-3. Story**

권장 10~15개  
구성:

*   창간사
*   발간 목적
*   K-문명의 가능성
*   대표 인터뷰 2개
*   문체부 브리핑 2개
*   입법부 브리핑 2개
*   지역문화 2개
*   사람&현장 2개

이는 실제 웹진 구조 요구와 일치한다.

**5\. Trust / Edge Case Fixture**

반드시 아래 edge case를 포함한다.

*   reviewer 없는 draft answer
*   evidence 없는 review answer
*   related story 없는 public answer
*   based on answers 없는 review story
*   자료 없는 종료 event
*   gallery만 있고 video 없는 event
*   broken CTA membership plan
*   correction pending story
*   duplicated slug candidate
*   long-stale public answer

AIO-PR의 린트 규칙과 B-SSoT Trust Layer를 실제 QA에 반영하려면 이런 실패 케이스 fixture가 필요하다.

**9\. Seed 생성 규칙**

*   모든 객체는 stable id 사용
*   slug는 최종 라우트 체계 반영
*   reviewer/expert는 실제 연결 가능해야 함
*   topic-answer-story-event 그래프가 끊기지 않게 생성
*   dummy 텍스트라도 구조는 실제 필드 길이를 반영
*   fixture마다 seed\_source, seed\_purpose 메타를 넣는 것을 권장

**10\. 최종 결론**

34번 문서는 개발과 QA를 가속하는 실행용 데이터 계약이다. 핵심은 문화강국네트워크의 실제 퍼블릭 구조와 운영 구조를 대표하는 최소 시드 세트를 만드는 것이다. Topic 4~6개, Answer 12~18개, Story 10~15개, Event 5~8개, Documents 10개+, Participation fixture와 Workflow/Trust edge case를 함께 넣어야 퍼블릭 UX, Admin Workflow, Feed/Sitemap, Observatory, Fix-It까지 전부 검증할 수 있다. 이는 문화강국네트워크 요구안의 웹진·행사·회원·자료 구조와 AIO-PR의 최소 런칭 스택 권고에 직접 부합한다.
