현재 첨부한 **GNB 행사/일정 페이지**는 “기록과 행사”라는 이름 아래에 **일정, 아카이브, 영상, 발표자료, 사진**이 한 화면에 느슨하게 섞여 있어, 사용자가 **무엇을 찾으러 왔는지에 따라 탐색 경로가 불명확**합니다. 반면 기존 SSoT에서는 이 메뉴를 **행사·토론회 허브**로 보고, LNB를 **전체 일정 / 지난 행사 / 순회토론회 / 특별 행사·전시**로 나누는 방향이 이미 정리돼 있습니다. 또한 토픽/택소노미 가이드상 이런 페이지는 단순 리스트가 아니라 `category`, `series`, `topic` 파라미터를 읽어 **동적 허브(Hub) 페이지**로 동작해야 합니다.  

또 실제 행사 자산은 충분히 강합니다. 연혁과 행사 목록에는 1차부터 11차까지 토론회, 지역문화대전환 I~IV, 광복 80주년 미술 전시회, 후원의 밤이 이미 정리되어 있어, 이 페이지는 단순 일정표가 아니라 **문화강국네트워크의 정책 공론장 아카이브**로 승격할 수 있습니다.  

아래처럼 재설계하는 것을 권합니다.

# 1. 이 페이지의 역할 재정의

## 페이지 한 줄 정의

**문화강국네트워크의 행사·토론회·기록·영상 자산을 구조적으로 탐색하는 공식 아카이브 허브**

## 페이지 목표

1. 사용자가 “예정된 행사”, “지난 토론회”, “순회토론회 시리즈”, “자료/영상” 중 원하는 것을 빠르게 찾게 한다
2. 행사를 단순 일정이 아니라 **정책 자산**처럼 보이게 한다
3. 행사 → 리포트 → 정답카드 → 브리프 → 영상으로 이어지는 연결 구조를 만든다
4. 향후 `topic / category / series` 기반 동적 허브로 쉽게 확장되게 한다

# 2. 현재 화면의 핵심 문제

## 1) 페이지명과 실제 기능이 어긋납니다

현재 제목이 **“기록과 행사”**인데, 사용자는 이게

* 전체 일정 페이지인지
* 아카이브인지
* 행사 기록 보관소인지
* 영상/자료 모음인지
  즉시 이해하기 어렵습니다.

## 2) 상단 탭이 너무 세분화되어 있습니다

현재 탭:

* 전체 기록
* 행사 일정
* 토론회
* 명사 영상
* 발표문/자료
* 사진 아카이브

이 구성은 데이터가 충분히 쌓인 뒤 2차 필터로는 좋지만, **1차 네비게이션**으로는 너무 세분화되어 있습니다. SSoT가 권장한 기본 LNB는 오히려 더 단순합니다. 

## 3) 대표 카드가 “의미”보다 “존재”만 보여줍니다

현재 카드에는 제목·날짜·장소는 있으나,

* 이 행사가 왜 중요한지
* 어떤 시리즈 소속인지
* 어떤 주제인지
* 결과 자료가 있는지
  가 보이지 않습니다.

## 4) “모집중”과 “기록물”이 한 그리드에 뒤섞여 있습니다

예정 행사와 과거 기록은 사용자 목적이 다릅니다.
같은 카드 그리드에 섞이면 정보 탐색 효율이 떨어집니다.

## 5) 색상 대비와 정보 강조가 약합니다

Hero는 진한 네이비지만, 아래 카드 영역은 거의 같은 밝기의 회색/흰색 박스가 반복되어 시각적 리듬이 약합니다. 카드의 상태 배지, 시리즈, 결과물 유무가 더 선명하게 보여야 합니다.

### 상단 설명

1. Hero

3. 필터 바

## 5-2. 지난 행사

### 목적

종료된 행사를 **정책 자산 아카이브**로 재구성

### 상단 설명

지난 행사는 단순 기록이 아니라 정책 자산입니다. 문화강국네트워크가 진행한 토론회와 행사의 주제, 핵심 논점, 후속 과제를 구조적으로 다시 확인할 수 있도록 정리합니다. 이 페이지는 연혁 기반 카드형 구성으로 충분히 만들 수 있습니다. 

### 초기 대표 카드 예시

* [1차] 문화산업의 위기, 문화정책의 위기
* [2차] 문화정책의 방향과 지속 가능성
* [3차] 문화산업과 문화가치, K-다움
* [4차] 인공지능과 문화산업의 현장
* [10차] 한식의 세계화, K-푸드가 이끄는 국가 경쟁력
* [11차] 기초예술 진흥과 예술인 복지 – 국가 시스템 구축 필요 

### 추가 구성

* 연도 필터
* 주제 필터
* 결과 자료 보유 여부 필터
* 하단 “최근 리포트로 이어보기”

## 5-3. 지역문화대전환 순회토론회

### 목적

6~9차를 하나의 시리즈 허브로 승격

### 상단 설명

문화강국네트워크는 지역문화 생태계, 문화자치, 지역 정주와 콘텐츠 순환, 지역문화 진흥 과제 등을 주제로 순회토론회를 진행해 왔습니다. 고흥, 전북, 거제, 제주 등 지역 현장을 중심으로 지역문화의 방향과 과제를 논의했습니다. 

### 시리즈 대표 카드 4개

* I. 지역문화 생태계, 현장을 진단하다
* II. 문화자치, 지역이 만든다
* III. 사람이 머무는 지역, 콘텐츠로 순환하는 국가
* IV. 대전환의 시대, 지역문화 진흥의 주요 과제 

### 추가 구성

* 지도형 탐색(고흥/전북/거제/제주)
* 회차별 핵심 질문
* 관련 기사 연결
* 관련 정답카드 연결
* 하단 “지역문화·문화자치” 토픽으로 더 보기

이 시리즈는 실제 행사 목록에서도 연속성 있게 존재하므로 별도 허브가 충분히 가능합니다.

## 현재 문제

## 개선 원칙

* 진한 Navy 유지
* 포인트는 Cyan 1개만 사용

### 본문

# 10. CMS/데이터 관점 반영 포인트

현재 설계상 이 페이지는 `topic / category / series`를 읽는 동적 허브가 되어야 하므로, 프론트는 “레이아웃 틀”, CMS는 “분류와 관계”를 담당하는 구조가 적절합니다. 가이드도 네비게이션과 콘텐츠 분류를 분리하고, 쿼리 파라미터로 허브를 만들라고 설명합니다.

## 권장 파라미터 예시

* `/events`
* `/events?category=forum-report`
* `/events?series=regional-culture-transition`
* `/events?type=video`
* `/events?asset=pdf`
* `/events?topic=local-culture-autonomy`

# 12. 한 줄 결론

**현재 행사/일정 페이지는 ‘기록과 행사’라는 모호한 허브에서 벗어나, ‘행사·토론회’라는 명확한 공식 아카이브 허브로 재정의하고, 1차 LNB는 `전체 행사 아카이브 / 지난 행사 / 지역문화대전환 순회토론회 / 특별 행사·전시`, 2차 필터는 `일정 / 토론회 / 영상 / 발표자료 / 사진`으로 분리하는 것이 가장 적절합니다. 이렇게 해야 문화강국네트워크의 행사 자산이 단순 일정표가 아니라 정책 공론장 아카이브로 보이게 되고, 향후 topic·series·category 기반의 동적 CMS 구조와도 자연스럽게 맞물립니다.**  

원하시면 다음 단계로
**이 페이지를 프론트엔드 개발에 바로 넘길 수 있도록 컴포넌트 단위 와이어프레임 표**로 바꿔드리겠습니다.
